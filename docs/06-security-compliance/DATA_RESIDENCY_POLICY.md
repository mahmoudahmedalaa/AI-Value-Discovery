# Data Residency Policy

Client data must remain inside the approved deployment boundary. This includes uploaded files, extracted text, embeddings, prompts, model outputs, scores, reports, audit logs, and backups.

Deployment modes: demo, managed single tenant, client hosted, restricted.

Each workspace defines allowed providers, regions, prompt/output movement, embedding generation boundary, and logging policy.

Documents must be classified: public, internal, confidential, regulated, restricted.
