# Coding Standards

Type-safe code, clear naming, no hardcoded secrets, no hidden external dependencies, no cross-tenant assumptions, server-side validation, testable business logic, small modules.

TypeScript strict mode. Use schema validation. UI must include loading/empty/error states and accessible labels. AI output must be schema-validated and policy-checked.

Commits: feat, fix, docs, test, refactor, chore, ci.
